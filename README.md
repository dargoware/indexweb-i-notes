# indexweb-i-notes

## 简介

indexweb-i-notes 是一个用于归档和发布独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为一系列独立页面的集中存储与索引。

这些页面可能包括笔记、工具界面、信息展示页、小型演示页面等，均以纯 HTML 形式存放，便于直接访问或嵌入。

## 目录结构

```
indexweb-i-notes/
├── pages/          # 存放所有独立 HTML 页面
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/         # 页面所需的静态资源（CSS、JS、图片等）
├── index.html      # 页面索引列表（可选）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件为一个独立页面，命名以 page-xxx 形式，方便识别与引用。
- `assets/`：存放页面共用的或专用的样式、脚本、字体、图标等资源。
- `index.html`：提供所有页面的快速导航（如存在）。

## 页面归档说明

- 每个页面均为完全自包含的 HTML 文件，或通过相对路径引用 `assets/` 中的资源。
- 页面内容不涉及任何商业推广、外部链接跳转或敏感信息。
- 页面可能随时间更新或移除，历史版本可通过 Git 提交记录追溯。
- 若页面依赖外部资源（如 CDN 库），会在文件头部注释说明。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，按顺序命名（如 page-xxx.html），并更新 `index.html`（如有）。
- 更新页面：直接修改对应文件，建议在提交信息中简要说明变更内容。
- 删除页面：移除文件，并相应更新索引（如有），不建议删除历史提交记录中的文件。
- 资源管理：若页面使用专用资源，建议在 `assets/` 下按页面建立子目录，避免冲突。

## 使用建议

- 可通过 GitHub Pages 将此仓库发布为静态站点，直接访问各页面。
- 也可将页面下载到本地或自建服务器使用。
- 欢迎提交 Issue 或 Pull Request 提出改进建议。
